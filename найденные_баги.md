## BUG-01: Авторасстановка для поля 5x5 ставит только 1 корабль

**Критичность:** Высокая  
**Связь с требованиями:** КРО №3, КРО №4

**Шаги воспроизведения**
1. Авторизоваться как `LuckyAlbert` / `123`.
2. В лобби выбрать конфигурацию `Прибрежные воды` (5x5).
3. Нажать кнопку авторасстановки.

**Ожидаемый результат**
- Расставлены все корабли для 5x5: `2x1-палубных`, `1x2-палубный`, `1x3-палубный` (в сумме 7 клеток).
- Кнопка `На абордаж!` активна.

**Фактический результат**
- На поле остается только `1` занятая клетка.
- Кнопка `На абордаж!` остается неактивной.

**Подтверждение**
- Проверка через UI и JS-матрицу: `occupied_cells = 1`, `can_create = false`.

## BUG-02: Для поля 10x10 матч стартует с таймером 5 минут вместо 10

**Критичность:** Высокая  
**Связь с требованиями:** КРО №3, КРО №6

**Шаги воспроизведения**
1. Игрок 1 создает игру с конфигурацией `Неизведанные глубины` (10x10).
2. Игрок 2 присоединяется к игре.
3. Сделать первый выстрел.

**Ожидаемый результат**
- Для 10x10 стартует длительность игры `10 минут` (`600` секунд), таймер около `09:59`.

**Фактический результат**
- В сообщении `FIRE_RESULT` приходит `gameDuration = 300`.
- UI таймер показывает `04:59`.

**Подтверждение**
- Перехват WebSocket: `type=FIRE_RESULT`, `gameDuration=300`.

## BUG-03: В UI победитель определяется наоборот (инверсия winner)

**Критичность:** Критическая  
**Связь с требованиями:** КРО №8, КРО №9

**Шаги воспроизведения**
1. Запустить матч между `LuckyAlbert` и `LuckyAlbert2`.
2. В процессе матча игрок `LuckyAlbert2` выходит из игры (кнопка `Вернуться к выбору игры`).
3. Проверить событие завершения и текст победителя у `LuckyAlbert`.

**Ожидаемый результат**
- Победа присваивается оставшемуся игроку (`LuckyAlbert`).
- UI показывает победителем `LuckyAlbert`.

**Фактический результат**
- WebSocket `GAME_FINISHED` содержит `winner = LuckyAlbert`.
- UI показывает победителем `LuckyAlbert2` (`Победил(а) LuckyAlbert2!`).

**Подтверждение**
- Несоответствие между данными WebSocket и отображением на экране.
- Дополнительно подтверждено в сценарии завершения по таймеру: в `GAME_FINISHED` приходит `winner=LuckyAlbert`, UI показывает победителем `LuckyAlbert2`.
